# Бот-викторина «Какое у вас тотемное животное?»

Telegram-бот для популяризации программы Московского зоопарка [«Возьми животное под опеку»](https://moscowzoo.ru/my-zoo/become-a-guardian/). 
Пользователь проходит короткую викторину (по своему содержанию ориентированную, преимущественно, на детей), 
основанной на реальных фактах об обитателях зоопарка, в финале получает соответствующее своему характеру (насколько мои познания психологии позволяют xD) 
тотемное животное и предложение поддержать его через программу опеки.

## Быстрый старт (для проверки работоспособности бота)

Чтобы запустить бота локально, нужно создать **своего** бота в Telegram (токен пользователя в репозиторий не попадает) и заполнить `.env`. Шаги:

**1. Получить бот-токен у [@BotFather](https://t.me/BotFather)**

В Telegram:

- открыть [@BotFather](https://t.me/BotFather), написать `/newbot`;
- придумать имя бота (любое) и username (обязательно заканчивается на `bot`, например `my_zoo_quiz_bot`);
- BotFather пришлёт длинную строку вида `7849234234:AAH...rA9o` — это и есть `BOT_TOKEN`.

**2. Узнать свой числовой Telegram-id** у [@userinfobot](https://t.me/userinfobot) — это понадобится, чтобы бот пересылал вам результаты викторины (демонстрация контактного механизма).

**3. Заполнить `.env`**

```bash
cp .env.example .env                 # Windows: copy .env.example .env
```

> **Если в Finder/Проводнике не видно файлов `.env.example` и `.gitignore`** — это потому что они начинаются с точки и считаются скрытыми. 
> На macOS показать их можно сочетанием `Cmd + Shift + .` в Finder. 
> На Windows — поставить галку «Скрытые элементы» во вкладке «Вид». 
> В коде-редакторе (VS Code, PyCharm) скрытые файлы видны сразу.

После копирования рядом с шаблоном появится отдельный файл `.env` — это и есть рабочий файл с настройками, который надо заполнить (сам шаблон `.env.example` не трогаем). Откройте `.env` любым редактором и впишите значения:

| Переменная       | Куда вставить |
| ---------------- | ------------- |
| `BOT_TOKEN`      | токен из шага 1 |
| `BOT_USERNAME`   | username бота без `@` (нужен для ссылок «поделиться») |
| `ADMIN_CHAT_ID`  | ваш числовой id из шага 2 |
| `ADMIN_USERNAME` | любой username для демонстрации, можно ваш |
| `ADMIN_EMAIL`    | любой e-mail для демонстрации |
| `ADMIN_PHONE`    | любой телефон для демонстрации |

В коде ничего вручную править не нужно — все секреты только в `.env`.

**4. Запустить**

**Необходимо открыть терминал по адресу корневой папки бота и выполнить**
```bash
python run.py
```
(на Windows — также `py run.py`).

**ВАЖНОЕ ПРИМЕЧАНИЕ!!! На территории РФ может не сработать без VPN и тогда придется сделать всё в ручную (ниже приведена инструкция)**

`run.py` сам создаст виртуальное окружение `.venv`, доустановит туда зависимости из `requirements.txt` и запустит бота. Работает из любого терминала, в том числе при активной conda `base` — поэтому даже после перезапуска терминала достаточно той же команды `python run.py`, ничего активировать вручную не нужно.

> **Хочется собрать venv вручную?** Можно и так:
> ```bash
> python -m venv .venv
> source .venv/bin/activate            # Windows: .venv\Scripts\activate
> pip install -r requirements.txt
> python bot.py
> ```
> Эффект тот же. Собрав так один раз, до перезапуска компьютера или терминала больше подобных танцев с бубном не потребуется. Бдует достаточно в терминале написать команду python bot.py.
> Соответственно, после каждого нового вызова терминала придётся помнить о `source .venv/bin/activate`. Поэтому лучше использовать `python run.py`.

В консоли должно появиться `Бот запущен`. Закрывать терминал не нужно — пока он открыт, бот работает. Остановить — `Ctrl + C`.

**5. Проверить в Telegram**

Открываете созданного бота → пишете `/start` от своего личного аккаунта и далее проходите викторину. 
Этот первый `/start` важен: Telegram запрещает ботам писать пользователю первыми, и без этого шага не сработает кнопка «Связаться» и пересылка отзывов в Ваш чат.

## Логика викторины

1. **Стартовый экран** (`/start`) — приветствие, три кнопки: «Начать викторину», «О программе опеки», «Помощь».
2. **Вопросы** — 10 штук, по 4 варианта в каждом. Содержание и веса задаются в `data.py` - `QUESTIONS`. Порядок вариантов перемешивается случайно при каждом показе.
3. **Алгоритм результата** — каждый вариант ответа добавляет очки сразу нескольким животным. В финале побеждает то животное, у кого сумма больше. При равенстве баллов выигрывает то животное, которое идёт раньше в словаре `ANIMALS` (порядок словаря в Python).
4. **Финальный экран** — фото животного, его имя, короткий «титул», описание характера (насколько хватило моего скилла психолога xD), факт о виде, мини-интро о программе опеки. Дальше кнопки: «Узнать о программе опеки», «Поделиться», «Ещё раз», «Связаться», «Оставить отзыв».

В `data.py` восемь животных: манул, амурский тигр, капибара, сурикат, дикобраз, фламинго, слон, лев. Каждое можно получить «правильным» набором ответов.

## Реализация задач, поставленных в итоговм задании

| # | Критерий                       | Где реализовано |
| - | ------------------------------ | --------------- |
| 1 | Запуск бота                    | `bot.py`, `bot.infinity_polling()` |
| 2 | Взаимодействие с пользователем | приветствие, дружелюбный тон, `/help`, меню команд |
| 3 | Модуль викторины               | вопросы и варианты — в `data.py`, легко править |
| 4 | Алгоритм обработки ответов     | взвешенная система баллов в `cb_answer` + `finish_quiz` |
| 5 | Подведение результатов         | фото + имя + характер + факт + блок про опеку |
| 6 | Изображения                    | реальные фотографии; отправка с тройным fallback (file_id → URL Wikimedia → локальный файл) |
| 7 | Соцсети                        | Telegram, ВКонтакте, Twitter/X, копия текста; везде ссылка на бота |
| 8 | Контактный механизм            | контакты сотрудника + автопересылка результата викторины админу |
| 9 | Перезапуск                     | кнопка «🔁 Ещё раз», команды `/start` и `/quiz` |
| 10 | Обратная связь                | `/feedback` или кнопка → CSV (`feedback.csv`) + копия админу |
| 11 | Сопровождение                 | прогресс «N / N», `/help`, реакция на любой ввод |
| 12 | Креативность                  | 8 уникальных характеров, юмор, реальные факты о зоопарке |

Дополнительно реализован функционал:

- **Конфиденциальность.** Команда `/privacy` объясняет, какие данные собираются. Все секреты в `.env` (исключён из git). Бот не запрашивает телефон, e-mail и тд.
- **Масштабируемость.** `threaded=True` в TeleBot позволяет одновременно обслуживать нескольких пользователей; состояние хранится в `dict` в памяти.
- **Мониторинг.** Логирование в `bot.log` и в консоль; ошибки сетевых вызовов перехватываются и не валят бота.

## Заметки по реализации

- **Архитектура.** pyTelegramBotAPI (синхронный)
- **Состояние пользователя** хранится в обычном словаре в памяти; для одного процесса этого достаточно. 
- **Алгоритм.** Каждый вариант ответа добавляет очки сразу нескольким животным. Веса подбирались так, чтобы каждое из 8 животных можно было получить «своим» набором ответов (проверено перебором).
- **Длина ответов.** Все варианты ≤28 символов — на узких экранах Android кнопки переносятся, а более длинные обрезаются. 28 символов было подобрано опытным путем, в тестировании учавствовало около 10 человек.
- **Перемешивание вариантов.** Каждый раз показ вопроса со случайным порядком — `random.shuffle` поверх списка `(индекс, вариант)`. Индекс зашит в `callback_data`, поэтому очки начисляются стабильно.
- **Картинки.** Лежат локально в `images/` для надёжности. При отправке бот пробует три источника по очереди: `file_id` Telegram'а из кеша (мгновенно), URL Wikimedia (Telegram сам качает), локальный файл (резерв).
- **Конфиденциальность.** Никакие платёжные данные, телефоны и e-mail у пользователя не запрашиваются.

## Где при желании можно изменить содержание теста и другого контента

- **Вопросы и веса:** `data.py`, список `QUESTIONS`. Логика бота не меняется.
- **Описания животных и картинки:** `data.py`, словарь `ANIMALS`.
- **Тексты приветствия и блока программы опеки:** константы `WELCOME`, `ABOUT`, `PRIVACY` в `bot.py`.

## Возможные проблемы

- **`SSL: CERTIFICATE_VERIFY_FAILED`** на macOS при попытке докачать картинку. Решается командой `/Applications/Python\ 3.11/Install\ Certificates.command` (один раз). Если картинки уже лежат в `images/`, бот ничего скачивать не будет, и проблема не возникнет в принципе.
- **Бот не пересылает результаты в чат админа** — проверьте, что вы хотя бы раз написали боту `/start` от того аккаунта, который указан в `ADMIN_CHAT_ID`. Telegram запрещает ботам писать первыми.
- **Кнопки «Поделиться» не реагируют** — Telegram запрещает в inline-кнопках любые URL, кроме `http(s)://` и `tg://`
